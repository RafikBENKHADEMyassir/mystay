# Internationalization Plan

## Strategy
- Use Next.js built-in i18n routing with locales `en`, `fr`, `es` (configured in `frontend/next.config.mjs`); default is `en`.
- Keep locale definitions in `frontend/src/lib/i18n/locales.ts`; add dictionaries when copy is ready.
- Avoid hard-coded strings in components; export text from locale dictionaries once available.

## Implementation Steps
1) Add translation dictionaries per locale (`frontend/src/lib/i18n/dictionaries/{en,fr,es}.ts`) with typed keys.  
2) Create a locale-aware layout (`frontend/src/app/[locale]/layout.tsx`) and migrate pages under `[locale]/(experience)/...`.  
3) Add locale switcher component and persist choice (cookie or profile setting).  
4) Localize dates/numbers/currencies using `Intl` APIs; respect PMS currency per property.

- Verify RTL readiness if new locales require it.
